This lab examines the functions used to backup and restore data in MongoDB.

Back up a MongoDB database using mongodump

mongodump

The mongodump function is used to create a backup of a whole database (or a single collection). The mongodump command should be run in the system command prompt, not in the mongodb shell.

The command shown here will create a backup of the grades database.

mongodump --db grades

To create a backup of a single collection in a database, you would use the following command:

mongodump --db college --collection students

The mongodump function outputs a folder containing a BSON (binary JSON) file and a metadata file. You may need to specify an output folder for the function, especially on the college computers where you don't have write access for the C: drive. Use this syntax to specify an output folder:

mongodump --db grades --out G:\dump

Using mongorestore to restore a MongoDB database

We will now restore the backup we created to confirm that it was successful. First, we need to drop the grades database.

Drop the grades database

Execute the following commands from the mongo.exe client (one at a time)

show dbs
use grades
db.dropDatabase()
show dbs

Restore the grades database

Run this command from the system command prompt:

mongorestore --db grades G:\dump\grades
